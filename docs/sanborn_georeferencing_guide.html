Sanborn Map Georeferencing Guide

This guide walks through the process of georeferencing downloaded Sanborn fire insurance map sheets in QGIS, then converting them to PMTiles for display in the Interactive City Map. The overall pipeline is: 

Download (import_sanborn.py) → Georeference (QGIS, this guide) → PMTiles (georeference_to_pmtiles.sh)

Prerequisites

Tool Version Purpose Install
QGIS 3.28+ LTR Georeferencing UI qgis.org/download
GDAL 3.6+ Raster processing brew install gdal
pmtiles CLI latest Tile packaging brew install pmtiles

You also need Sanborn sheets downloaded via import_sanborn.py. See the backend README for instructions. Sheets will be in data/sanborn/ organized by decade.

Step-by-Step Georeferencing in QGIS

1

Open QGIS and Add a Basemap

Launch QGIS and add a modern basemap layer to use as your reference. This gives you the current street grid to align the Sanborn sheet against.

  1. Go to Web → QuickMapServices → OSM → OSM Standard. If you do not have QuickMapServices installed, go to Plugins → Manage and Install Plugins and search for it.
  2. Zoom to the area of Chicago your Sanborn sheet covers.
[Screenshot: QGIS with OpenStreetMap basemap zoomed to Chicago]
2

Open the Georeferencer

The Georeferencer is a built-in QGIS tool (no separate plugin needed in QGIS 3.26+).

  1. Go to Layer → Georeferencer (or Raster → Georeferencer in older versions).
  2. In the Georeferencer window, click Open Raster (the folder icon) and navigate to your downloaded Sanborn sheet image.
  3. The Sanborn sheet will appear in the Georeferencer canvas.
[Screenshot: Georeferencer window with a Sanborn sheet loaded]
3

Place Ground Control Points (GCPs)

Ground Control Points are pairs of coordinates: a point on the Sanborn sheet matched to its real-world location on the basemap. The more accurate your GCPs, the better the final alignment.

  1. Click the Add GCP Point tool (crosshair icon) in the Georeferencer toolbar.
  2. Click on an identifiable location on the Sanborn sheet (e.g., a street intersection).
  3. In the dialog that appears, choose "From Map Canvas", then click the matching location on the QGIS basemap.
  4. Repeat for at least 8–12 GCPs spread across the entire sheet.
GCP Strategy

Best GCP locations (in order of reliability):

  1. Street intersections — Most reliable. Streets rarely move. Use the center of the intersection.
  2. River/canal bends and bridges — Waterway geometry is stable over a century.
  3. Railroad crossings — Rail alignments persist even if tracks are removed.
  4. Major building corners — Use only for buildings you can confirm still exist (e.g., churches, city hall).

Spread your GCPs evenly across the entire sheet. Avoid clustering them in one area — this causes distortion in other parts of the image.

[Screenshot: GCP points placed on Sanborn sheet with matching basemap locations]
4

Choose the Transformation Type

The transformation type determines how the raster image is warped to fit the GCPs. Different types suit different situations.

Type Min GCPs Best For Notes
Thin Plate Spline 3+ Historical maps with irregular distortion Recommended — handles local warping well
Polynomial 1 (Affine) 3 Minimal distortion, just shift/rotate/scale Good for maps that are already close to correct
Polynomial 2 6 Moderate curvature correction Middle ground; fewer GCPs needed than TPS
Polynomial 3 10 Significant warping Needs many accurate GCPs to avoid overfitting
Recommendation

For Sanborn maps, Thin Plate Spline (TPS) is almost always the best choice. These maps were hand-drawn and have irregular distortion patterns that TPS handles well. Use at least 8 GCPs for good results, 12+ for excellent results.

To set the transformation type:

  1. In the Georeferencer, go to Settings → Transformation Settings.
  2. Set Transformation type to Thin Plate Spline.
  3. Set Target CRS to EPSG:4326 (WGS 84).
  4. Set Resampling method to Cubic for the best visual quality.
5

Configure Output Settings

Still in the Transformation Settings dialog:

  1. Set Output raster to your desired path, e.g.:
    data/sanborn/georeferenced/1890s/sheet_001_geo.tif
  2. Check Create world file (optional but useful for debugging).
  3. Check Load in project when done to immediately see the result overlaid on the basemap.
  4. Set Compression to JPEG to keep file sizes manageable.
  5. Click OK.
[Screenshot: Transformation Settings dialog with recommended values filled in]
6

Run the Georeferencing

  1. Review your GCPs in the table at the bottom of the Georeferencer. Check the Residual column — values above 5–10 pixels indicate a misplaced GCP.
  2. Remove or adjust any GCPs with high residuals.
  3. Click the Start Georeferencing button (green play icon).
  4. QGIS will warp the image and save the output GeoTIFF.
Verify the Result

After georeferencing, the sheet should appear overlaid on the basemap. Toggle transparency to verify that streets align with their modern counterparts. If alignment is poor in some areas, go back and add more GCPs in those regions, then re-run.

7

Repeat for All Sheets in an Era

Each Sanborn sheet covers a small area (a few blocks). For a full neighborhood or city coverage, you need to georeference many sheets. Organize your output GeoTIFFs by era: 

data/sanborn/georeferenced/
  1890s/
    sheet_001_geo.tif
    sheet_002_geo.tif
    ...
  1905-1910/
    sheet_001_geo.tif
    ...
  1920s-1950/
    sheet_001_geo.tif
    ...

Converting to PMTiles

Once you have georeferenced GeoTIFFs for an era, run the conversion pipeline to produce a PMTiles file the frontend can display: 

cd backend/scripts

# Process the 1890s era
./georeference_to_pmtiles.sh \
  ../../data/sanborn/georeferenced/1890s \
  ../../tiles \
  "1890s"

# Process 1905-1910
./georeference_to_pmtiles.sh \
  ../../data/sanborn/georeferenced/1905-1910 \
  ../../tiles \
  "1905-1910"

# Process 1920s-1950
./georeference_to_pmtiles.sh \
  ../../data/sanborn/georeferenced/1920s-1950 \
  ../../tiles \
  "1920s-1950"

The script will:

  1. Convert each GeoTIFF to a Cloud Optimized GeoTIFF (COG)
  2. Build overview pyramids for fast multi-zoom rendering
  3. Generate a tile pyramid using gdal2tiles.py
  4. Package tiles into a single PMTiles file per era

The output PMTiles files go into the tiles/ directory. Serve them with any static file server, or place them where the frontend can access them. Update the tile URLs in frontend/src/map/controls/EraSelector.tsx if needed.

Tips for Accuracy

  • Work at high zoom. Zoom in closely when placing GCPs on both the Sanborn sheet and the basemap. Pixel-level accuracy at intersections makes a big difference.
  • Use "From Map Canvas" mode for GCP coordinate entry rather than typing coordinates manually. Visual matching is faster and less error-prone for this use case.
  • Save your GCPs. In the Georeferencer, go to File → Save GCP Points. This creates a .points file so you can reload and adjust later.
  • Check residuals before running. A GCP with residual above 10 pixels is probably wrong. Delete it or fix it.
  • Use multiple basemap sources. If OSM does not show enough detail, try a satellite layer or the Chicago 1939 aerial photos for cross-reference.
  • Batch similar sheets. Sheets in the same volume often have consistent distortion. After georeferencing a few, you will develop an intuition for where GCPs need to go.
  • Name files consistently. Use the sheet number from the Sanborn atlas (visible in the corner of each sheet) in the filename. This makes the manifest and file organization much easier to navigate.

Troubleshooting

Streets are aligned in one area but shifted in another

Add more GCPs in the misaligned region. With Thin Plate Spline, local corrections do not affect areas that are already well-aligned.

The output GeoTIFF has black borders or NoData areas

This is normal — the warped image does not perfectly fill a rectangular bounding box. The georeference_to_pmtiles.sh script handles this. If you need to set a specific NoData value: 

gdal_translate -a_nodata 0 input.tif output.tif

The Georeferencer crashes on large images

Sanborn TIFFs from the LoC can be very large. Try downsampling first: 

gdal_translate -outsize 50% 50% large_sheet.tif smaller_sheet.tif

Georeference the smaller version, then apply the same GCP points file to the full-resolution image.

EPSG:4326 vs EPSG:3857

Always georeference to EPSG:4326 (WGS 84 geographic coordinates). The tile generation pipeline handles reprojection to Web Mercator (EPSG:3857) automatically.
