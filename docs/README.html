Interactive City Map

Multi-layer urban map with buildings, zoning, transit, historical Sanborn overlays, and interior building maps.

Open Source MIT License Multi-City

Project Description

Interactive City Map is an open-source web application that renders rich, multi-layer maps of American cities. It combines modern geospatial data (buildings, zoning, points of interest, railroads, transit lines, waterways, parks) with historical Sanborn fire insurance map overlays and detailed interior building maps — including AI-extracted floor plans, venue-scraped layouts, user-contributed uploads, and OpenStreetMap indoor data.

The project starts with Chicago as the primary city, with support for Dayton OH, Detroit MI, and Cincinnati OH following via YAML-driven multi-city configuration.

Features

Map Layers (20+)

  • Buildings with footprints and metadata
  • Zoning districts
  • Points of interest (POIs)
  • Railroads
  • Transit routes and stops
  • Waterways
  • Parks and green spaces
  • Streets and address parcels

Sanborn Historical Overlays

  • Georeferenced Sanborn fire insurance map overlays
  • Crop and align historical maps to modern geography
  • Opacity and blend controls

Interior Building Maps

  • Floor plans from multiple sources: Sanborn crops, venue scraping, user uploads, OSM indoor data
  • AI floor plan extraction using OpenCV and Tesseract OCR
  • Interactive interior editor — draw walls, rooms, and features
  • Floor level selector for multi-story buildings
  • Source selector with confidence scoring
  • Community contributions with verification and moderation

Search & Interaction

  • Typeahead search powered by PostgreSQL full-text search + trigram similarity
  • Building-level detail panels with click interaction
  • Measurement tool for interior distances

Multi-City Support

  • YAML-driven city configuration (data sources, bounding boxes, layer toggles)
  • Chicago (primary), Dayton OH, Detroit MI, Cincinnati OH

Quick Start

Prerequisites

  • Docker & Docker Compose
  • Python 3.12+
  • Node.js 20+

1. Clone and start infrastructure

git clone https://github.com/your-org/interactive-city-map.git
cd interactive-city-map
docker compose up -d        # PostgreSQL+PostGIS, Martin, MinIO

2. Backend setup

cd backend
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
alembic upgrade head
python -m app.data_import.run --city chicago
uvicorn app.main:app --reload --port 8000

3. Frontend setup

cd frontend
npm install
npm run dev                  # http://localhost:5173

Architecture

The application follows a layered architecture with clear separation of concerns:

  • Routes — FastAPI endpoint handlers that validate input and delegate to services
  • Services — Business logic layer (building service, interior service, search service, floor plan extraction, venue scraping, geocoding, georeferencing)
  • Models — SQLAlchemy ORM models mapped to PostGIS-enabled PostgreSQL tables (buildings, parcels, POIs, railroads, transit, waterways, zoning, interior rooms/walls/features/sources)
  • Schemas — Pydantic schemas for request/response validation
  • Adapters — Swappable external service integrations (geocoder, POI fetcher, S3-compatible storage via MinIO)
  • Data Import — Modular importers for buildings, parcels, POIs, railroads, transit, waterways, zoning, Sanborn maps, and interior sources (OSM indoor, venue scraping, county records, user uploads)

The frontend uses React 18 with MapLibre GL JS for GPU-accelerated map rendering. Vector tiles are served by Martin directly from PostGIS. Interior map components include an interactive editor (draw walls/rooms/features), floor level selector, source selector with confidence scoring, measurement tool, and raster/vector/label layers.
